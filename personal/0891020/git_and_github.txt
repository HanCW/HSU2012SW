git 장점
영리목적 서비스와 오픈소스를 위한 무상서비스가 모두 제공되기 때문에 자기의 목적에 맞게 사용가능하다.
자신의 소스(Data)를 웹상에 올려서 사람들과의 공유가 가능하기 때문에 사람들과의 피드백을 통한 소스의 개선이 가능하다.
또한 웹상에 올리기 때문에 분실의 위험도 적다.
그리고 자신의 프로젝트를 git에 올림으로써, 과거의 자신이 어떤 프로젝트를 했었고, 어떤 코드를 사용해서 작성했었는지 알기가 쉽다.

git 사용후 느낀점
우선 git을 처음 알게되어서 생소했다.
그리고 설치하는 과정에서 무척 애를 먹었다. 생소해서 readme 파일을 잘못건드리는 바람에....그래서 git 설정을 2~3번 하다보니 이제 설치.설정은 잘할 수 있을 것같다.
처음에는 왜이렇게 번거롭게 과제를 올려야 하나 싶었는데, 하다보니까 이렇게 과제를 올리는것도 좋은것 같다는 생각이 들었다.
적어도 분실의 위험은 없을테니까 말이다.
과거에 진행 했던 과제들은 pc포맷이나, usb포맷으로 인하여 과제의 대부분을 잃어버렸는데 git을 사용해서 data를 관리하면 내가 했던 과제들을 모두 모아 볼수 있어서 좋을 것같아는 생각이 들었다.
