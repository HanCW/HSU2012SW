HTML5가 무엇인가?
그냥 HTML5란 무엇인가 말하자면 HTML5란 HTML의 버전업이다.
버전업이니 많은 기능이 향상되고 더 좋아진 HTML이 HTML5이다.
고로 기존의 HTML에서 불편했던 점들을 보안하고 더 많은 기능을 지원한다.
처음 HTML이 만들어졌을때 그때의 수준으로는 온라인상으로 볼 수 있는게 지금에 비해 많이 적었다.
그때당시에는 그림이나 표정도였지 지금처럼 화려하지가 않았다.
하지만 웹사이트가 발전이되고 더 화려해지고 더 복잡해지기 때문에
HTML도 점점 새로운 버전이 나오게 될 수밖에 없다.
HTML4버전 후속으로 나오는것이 HTML5이다.
기존의 HTML에서는 음악 또는 동영상 같은 것들을 구현하기 위해 플러그인 이나 액티브X 같은 것들을 사용해야 하기 때문에
많은 문제점이 있었다.
하지만 HTML5는 이런것들을 내부에서 해결해 기존의 문제점들을 해결 할 수 있다.

HTML5는 왜 출현했는가?
HTML은 1990년대 초에 처음으로 나와 한해에 3번 버전업이 될만큼 활발했었다.
하지만 곧 암흑기에 들어갔고, W3C에서 차세대 웹언어를 개발해 서로 다른 표준으로 인해 각 브라우저에 맞는
코딩이 필요했기 때문에 HTML표준안에 대해 여러 토론이 오갔다.
W3C와 의견차이로 애플, 오페라 등은 따로 2006년 6월에 WHATWG라는 그룹을 만들어 차기 HTML을 만들기 시작한다.
이것은 웹 개발자들이 웹 어플리케이션 개발의 편의를 위한 것이였고, 이것은 점점 성공하기에 이른다.
반면 W3C는 XHTML 전환에 실패하여 새로운 HTML작업을 시작하고, WHATWG는 새로운 작업에 합류한다.
그래서 HTML표준을 HTML5라 명하고 WHATWG에서 작업하던 대부분의 표준안을 그대로 수용하여 개발하게 된다. 

HTML5의 현재 상태는?
아직까지는 HTML과 HTML4.01이 거의 모든 웹브라우저를 지원하고 있다.

HTML5의 미래는?
곧 머지않아 HTML5가 표준화가 되어 모든 웹브라우저를 지원할 것으로 보인다.

HTML5의 주요기능은?
HTML5는 기존의 HTML의 비해서 프로그래밍적인 요소들이 많이 들어가게 되었다.
HTML5 = HTML + CSS + Web API 로 보아도 무관할 것 같다.
1.캔버스 - 웹에서 즉시모드로 선이나 도형, 글자 등을 사각형에 그려 이미지 효과를 줄 수 있다.
2.웹 비디오 - 브라우저에서 직접 컨테이너, 코텍을 직접 지원할 수 있다.
3.오프라인 웹 응용 - 인터넷 연결이 지원되지 않아도 URL에 있는 리소를 PC로컬에 캐싱할 수 있다.
4.웹 워커 - 쓰레딩이 가능하다.
5.로컬저장소 - 로컬저장소를 지원한다.
6.웹 소켓 - 로컬저장소를 SQL로 직접 접근가능한 랩퍼를 제공한다.

등등 더 다양한 웹폼과 많은 기능들을 지원한다.
 

HTML5의 브라우저 지원상태는?
크롬, 사파리, 오페라 등에서 지원을 하고 있다.
