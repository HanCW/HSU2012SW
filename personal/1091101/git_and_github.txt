Git 장점 한학기동안 Git & GitHub 사용하며 느낀점 정리

 Git을 사용 하면서의 장점이란, 자기의 공간을 가지고 있고, 저장소에 올릴 수 있는
개개인이 관리하고 최종 산출물을 저장소에 올려서 저장 하는 것이다. 저장소에 올릴 때는
자기의 공간에서 복사본을 저장소에 올리는 것으로 버전이 바껴도 자기의 공간에서 
작업을 하여 수정본을 저장소에 올려서 버전관리가 용이 하다는 장점이 있는 것같다.
교수님은 Git과 친해 지라고 항상 매주 마다 과제를 내주시는데, 그 과제는 텍스트로
올리면서 푸쉬하고, 리케스트 받고 하는 과제다. 텍스트들이 소스 코드라면 완전한
Git을 사용하는 것이 되는 것 같다. 맨 처음에 Git을 소개 받았을 때는 신기하였다.
그리고 이 것을 내가 과연 잘 다룰 수 있을 까의 의문점도 있었지만, 어느 정도 푸쉬
그리고 확인하는 것 그리고 흐름이라든지 이런 것들을 볼 수 있는 안목이 생겼다.
다른 언어들, C++, 이클립스등의 연동, 블로그도 할 수 있다는 것을 수업이 끝나가는
시점에 알게 되었는데, 나중에는 Git을 사용하는 사람들이 많아 질 것만 같다.
그러나 나는 Git을 접해봐서 조금은 자신이 붙을 것같아 정말 좋은 것같다.
아직 더 Git을 더 사용해보아야 더 잘알 것같다. 팀프로젝트에서 조금 더 써보고
합작이 잘 나오기를 바라고 있다. 프로깃을 보면 이제 어느 정도는 보이기 때문에 
조금은 다행이라고 생각한다.
