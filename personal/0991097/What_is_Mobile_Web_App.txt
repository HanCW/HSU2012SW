HW#9 모바일웹

1.모바일웹

모바일웹은 일반적인 웹 기술로 개발되고 모바일 브라우저에서 실행되는 웹 애플리케이션이다.
모바일웹은 우리가 웹페이지를 이용할 때에 흔히 마주 대하는 HTML, CSS, 그리고 Jquery 등을 활용하여 브라우저에서 작동되는 모바일 환경을 의미한다.
기존 PC환경의 웹이 1024또는 1280의 해상도에 최적화되어 있기 때문에 모바일에서 보기에는 불편한 점이 많이 있어 모바일 해상도에서 최적화 되어 볼 수 있도록 만들어서 제공하는 것이다.
예) 아이폰과 안드로이드에 내장된 브라우저(웹 사이트,모바일사이트) 

2. 모바일앱

모바일 앱은 모바일 환경에서 구동할 수 있도록 만든 별도의 프로그램이다.
예) 아이폰이나 안드로이드폰에 제공되는 기본 어플이나 카카오톡, 푸딩카메라 등등의 모바일 응용프로그램.

3. 모바일웹앱 
웹브라우저에서 작동할 수 있도록 만든 앱이다.  
휴대용 무선 단말기로 인터넷에 접속하여 데이터 통신이나 인터넷 서비스를 이용하는 것이며, 모바일 브라우저에 의해 실행되는 웹 에플리케이션을 통칭한다.
예) 
바이크 : http://beta.dublinbikes2go.com   
문(비즈조선) : http://app.chosunbiz.com
게임 : http://psmon.x-y.net/bj/
 
4. 하이브리드앱
모바일웹과 모바일앱의 장점을 가져와서 혼합한 형태의 앱이다. 
항상 고정되어 있고 변하지 않는 요소들은 앱으로 만들어서 사전에 배포하고 변동되는 컨텐츠만 그때 그때 다운로드 받으면 되는 것이다. 
그렇기 때문에 앱을 계속적으로 업데이트하거나 추가 개발해야 하는 번그러움이 적다. 
환경적으로는 모바일앱과 동일한 환경을 가지고 있지만, 부분적으로는 HTML5 및 CSS를 사용하여 호환성이 확장된 것이다. 
예) 네이버 앱, 다음앱, 구글앱 등
(네이티브 앱의 형태로 앱스토어에서 배포되지만, 실제적인 구성을 보면 브라우저를 기반으로 하는 모바일 웹 형태로 구성되어 있는 것)

**모바일앱, 모바일웹, 모바일웹앱, 하이브리드앱 비교

모바일앱은 인터넷이 연결되어 있지 않아도 사용 가능하다는 장점이 있다. 
그러나 다른 방식과 비교했을 때 개발 기간과 비용이 많이 들어가며, 각기 다른 OS별로 개발해야 한다.
반면 모바일웹은 HTML 표준 방식으로 개발하면 OS에 상관없이 구현이 된다. 
모바일웹앱은 모바일앱에 비해 개발기간이 짧고 OS에 상관없이 웹브라우저로 접근이 가능하다.
그러나 인터넷을 통해 접속하므로 구동이 느리다.
하이브리드앱은 구동속도가 모바일웹보다 빠르지만, 자체 사이트만 지원하므로 타 사이트 브라우징이 불가능하다.
