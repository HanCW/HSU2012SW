Python
 파이썬에서의 가장 강점은 Introspection이다. 파이썬은 모든 것을 객체로 받아들이는데 이 Introspection은 이런 모든 객체들을 관리하며 조작하며 정보를 제공한다. 그래서 함수명 없이 정의하거나 호출하는 것이 가능하다.
  파이썬의 또 다른 장점으론 상대적으로 타 언어에 비해 쉬워 빠른 학습에 용이하다는 것이다. 코드가 간단하며 쉽게 눈에도 잘 들어오기 때문에 파악하기가 좋다. 코드의 간결함으로, C언어나 C++, Java 같은 언어에서 같은 코드 내용이라도 그 코드의 라인수가 1/3 정도이며, 개발 시간도 대략 1/3 정도이다. 또한 인터프리트 언어이기 때문에 한줄 수행할 때 마다 에러가 나는지 실행이 되는지 바로 알아내어 개발 시간을 단축시켜준다. 그렇기 때문에 수학적인 측면에선, 같은 시간에 더 많은 코드를 생산할 수가 있어 파이썬은 높은 생산성을 낳는다. 
   
Ruby
 루비는 인터프리팅 스크립트 언어이다. 이 언어 또한 배우기 쉽고 속도가 빠르기 때문에, 생산성이 아주 좋다. 불필요한 코드는 최소화해주며 코드 작성을 자동화해주기에 개발을 단축시켜준다. 생산성이 가장 큰 장점이기에 아이디어가 나오는 즉시 개발에 들어갔을 때, 타 언어에 비해 상대적으로 빠르다는 것이다. 
     또한 루비는 MVC 아키텍쳐를 지원한다. MVC패턴은 세부분으로 역할을 나누어 처리하는데, Model, View, Controller 이다. Model은 Active Record에서, View는 Action View에서, Controller는 Action Controller에서 담당을 한다. 다른 언어 같은 경우 객체와 데이터베이스 간에 대해 설정을 해주는 코드를 따로 넣어야하지만, 루비는 규칙을 따르면 자동으로 설정을 해주기 때문에 코드 생산에 대한 비용을 절감할 수 있다. 

PHP
 PHP의 가장 큰 장점은 무료가 아닐까 생각한다. 웹에서 많은 문서 파일을 찾을 수 있으며 개발자들의 노하우 또한 얻기 쉽다. JSP, ASP와 달리 PHP는 OS인 Apache와 데이터베이스인 MySQL 모두 무료이다. 때문에 웹 프로그래밍을 위한 돈의 지출이 전혀 없다. 또한, PHP는 윈도우, 리눅스 등에 대한 환경에 전혀 구애받지 않고 모든 운영체제에서 동작할 수 있다. PHP는 데이터베이스와의 연동 또한 우수한데, 다른 언어와 달리 단 몇 줄의 코드로 MySQL과 쉽게 연결해 사용할 수 있으며 빠르다. PHP 역시 간결하며, 쉽게 전달할 수 있는 명령 형식이기에 빠른 코딩을 할 수 있어 생산성이 높다. 

Javascript
 자바스크립트는 HTML 안에서 직접 코딩하여 실행하기 때문에 작업 능률도 높고 효율적이다. 객체 지향적 언어이며 브라우저 인터프리터 기능에 의하여 실행이 되며, 운영체제의 제한이 없다. 클라이언트에서 실행되기 때문에 서버 부하를 감소시킨다. 그래서 역시 속도가 빠르며, 웹 서버에 대한 부담이 줄어든다. 문법은 자바에 기초하지만 보다 쉬우며, 내장 함수 와 객체를 사용하여 프로그래밍 하는데 있어서 용이하다. 한정된 객체와 메서드를 사용하여 쉽게 습득할 수가 있다.
