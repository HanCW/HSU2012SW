

python은 기존의 C,C++,JAVA에 비해 코드를 작성하는 길이가 삼분에 일 이하라서 개발시간도 삼분의 일 이하로 단축할 수 있다. 다른 언어들과는 다르게 객체지향적으로 잘 만들어진 언어이고 머릿속의 알고리즘을 실제 구현시 흡사하게 만들어 배우기도 싶다. 메모리 소모도 JAVA의 절반 정도 이고 전체 적인 수행 속도는 C나 C++보다는 느리지만 JAVA보다는 빠른 속도를 보여준다. 여기서 보여주는 python의 장점은 빠른 개발속도로 인한 높은 생산성이다. 같은시간에 많은 코드를 작성해 많은 프로그램을 만들 수 있는 python은 프로그래머들에게는 획기적인 프로그래밍 언어이다. 하지만 프로그래머에게는 장점이지만 사용자에게는 단점이 될 수 있는 실행 속도이다. 요즘 컴퓨터 성능이 많이 항상되어 큰 차이를 느낄 수 없지만 프로젝트가 커지면 커질수록 속도에 대한 문제가 대두 될 수 있다. 교수님이 말씀하신 것처럼 여러 프로그래밍 언어를 잘해야 한다는 의미가 다른 언어를 조사하면서 알게 됬는데 C,C++혹은 Java에서 만드는 것이 효율적인 부분이 있을 수 있고 pythion으로 만드는 부분이 효율이 높을 수 있다. 그러므로 적제 적소에 맞는 언어를 쓰는 것이 좋을 것 같다. 마지막으로 python의 가장 큰 장점은 무료!이다.

 

Ruby는 마츠모토 유키히로씨가 스트레스 없는 쉬운 언어로 만들기 위해 제작되었다.Ruby의 첫 번째 장점은 일단 배우기 쉽다. 조사해본 결과 python 역시 쉽다는 장점이 많이 거론 됬지만 그보다 구조에 대한 장점이 많이 거론 됬는데 Ruby는 구조적인 장점보다는 쉽다는 것에 많은 장점이 거론되었다. 프로그래밍 언어 순위로 java가 1위 c가 2위 c# 3위

C++이 4위 python이 8위 ruby 는 12위로 상위에 속한 언어이다. 그리고 다른 장점으로는 C++이나 java처럼 객체지향언어이다 그리고 간결함과 생산성을 강조한 동적인 오픈 소스 프로그래밍 언어라 많은 사람들이 쉽게 사용할 수 있다. 기존에 흑히 사용하는 언어들에 비해 문법이 매우 단순하고 직관적이라서 자연스럽게 읽고 쓰기하기 편한 프로그램이다.

  

PHP 1994년 Rasmus Lerdorf에 의해 처음 만들어 졌다. Zeev Suraski와 Andi Gutmans에 의해 개량 되어서 현재와 유사한 PHP 3.0이 나왔고 후에 5.0에서 객체지향을 추가하면서현재까지 꾸준하게 업그레이드 되고 있다. PHP는 인터넷을 통해 무료로 쉽게 다운받아서 사용할 수 있다. 그리고 윈도우 유닉스 리눅스 등 대부분의 운영체제(OS)에서 사용할 수 있으면 개발 환경이 웹서버 환경에 대해 독립적이라서 쉽게 다른 환경을 인식할 수 있다. 그리고 C를 기반으로 제작된 언어라 프로그래밍 기초가 되어잇는 사람은 누가나 쉽게 배울 수 있다. 또다른 장점으로는 클래스의 생성,자료 업로드나 이민지 파일의 동적 생성, PDF 파일의 생성 IMAP,SNMP,NNTP,POP3,HTTP 등의 방대한 함수를 자체적으로 지원해 웹 프로그램에 필요한 기능을 쉽게 제공해 준다.

   
JAVASCRIPT란 사용자들의 pc에서 실행되는 스크립트 언어를 말한다. 자바스크립트는 웹브라우저가 분석해서 실행을 하는데 한줄한줄 분석하는 인터프리터형식이다 이렇게 하면 텍스트 처리 문자열 일치여부 검색 치환등 기능이 뛰어나고 코딩도 쉽다. 인터프리터형 언어의 장점으로 다른 C언어처럼 컴파일 할 필요없이 코딩한 대로 바로바로 실행이 가능해 실행 속도나 최적화는 C같은 언어 보다는 떨어지지만 컴파일을 해줄 필요가 없어 디버깅 은 이런 언어에 비해 뛰어나다. 그리고 여러 OS 유닉스 윈도우 등 소스파일 인식이 쉬워서 자신에게 맞는 시스템을 사용할 수 있다. 그리고 html과 연동이 좋아서 html 소스 코드안에 그대로 끼워 넣어도 컴파일 과정없이 그대로 브라우저를 실행가능하다.

JAVASCRIPT역시 다른 언어에 비해 훨씬 배우기 쉽고 이해하기 쉽다. 장점이 있으면 단점도 있는데 자바 스크립트의 객체는 한정되어있어서 인터넷 게임같은 다양하고 고난이도 프로그램은 작성하기 힘들다.
