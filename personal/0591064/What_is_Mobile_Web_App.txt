1. 모바일 웹
데스크탑 인터넷 브라우저에서 실행되는 웹을 모바일에 최적화 하여 만든 것이 모바일 웹 이다. HTML, CSS, Javascript, JSP, PHP, ASP, ASP.NET 등 일반적인 웹 기술로 개발되고 모바일 브라우저에서 실행된다. 모바일 웹은 풀 브라우징 방식으로 페이지를 이동한다. 또한 AJAX를 사용하여 페이지의 일정 부분을 갱신 할 수 있다. 네이버와 같은 포털 사이트를 비롯하여 대부분의 웹 서비스 업체에서 모바일 웹 환경을 제공하고 있다.

2. 모바일 앱
네이티브 언어로 모바일 디바이스에 최적화하여 개발된 앱을 말하며 네이티브 앱 이라고도 한다. 대표적인 예로 안드로이드 SDK와 Java를 이용하여 개발된 앱과 IOS SDK와 Object C를 이용하여 개발된 앱이 있다. 이러한 개발 과정으로 미루어 보았을 때 모바일 디바이스의 운영체제에 독립적이지 못하다는 단점이 있다. 하지만 카메라나 센서와 같은 모바일 디바이스의 센서를 제어할 수 있으며 고성능의 그래픽 처리가 가능하여 게임과 같은 고성능 앱 개발에 유리하다.

3. 모바일 웹앱
웹과 앱의 합성어로, 모바일 웹과 모바일 앱의 특징을 고루 갖춘 앱을 말한다. 모바일 웹과 같이 일반적인 웹 기술로 개발되고 모바일 브라우저에서 실행되지만, 풀 브라우징 방식이 아닌 단일 페이지 모델을 사용하여 화면을 전환한다. 따라서 모바일 웹 보다는 실행 속도가 빠르다. 다양한 분야의 업체에서 자사를 홍보하기 위해 화려한 UI를 가진 웹앱을 내놓고 있는 추세이다.

4. 하이브리드 앱
보통 웹과 앱의 특징을 고루 갖추어 마켓에 내 놓아도 손색이 없을 법 한 모바일 웹앱을 폰갭이나 앱스프레소와 같은 소프트웨어를 통해 하이브리드 앱으로 변환한다. 하이브리드 앱은 모바일 웹앱과는 달리 모바일 디바이스의 하드웨어를 제어할 수 있고, 이로써 모바일 앱에 더욱 가까워진 결과물을 애플 앱스토어나 구글 플레이에 등록할 수 있다. 모바일 웹 -> 모바일 앱의 형태로 개발되어 모바일 운영체제에 독립적이며 네이티브 언어가 아닌 웹 기술로 개발되기 때문에 개발비용이 비교적 적어진다. GS숍, 우리은행, 멜론 등의 사례가 있다. 업데이트의 불편함이 없고 웹페이지보다 빠르다는 장점으로 주목받고 있다.
