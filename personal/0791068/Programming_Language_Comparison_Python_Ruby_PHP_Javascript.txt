 javascript 

 html보다 더 고급화된 웹프로그래밍 언어로 동적이고 실감있는 웹페이지를 구현가능
다양한 기능이 있으며 다른 언어보다 비교적 간단하여 이해하고 사용하기 쉬운 장점이 있음
Javascript는 인터프리터형쪽에 가까우며 C언어처럼 컴파일없이 실행가능
글을 처리하거나 문자열의 일치 여부, 혹은 검색이나 순서변경등의 능력이 뛰어나고 코딩도 용이
컴파일의 반복이 없어서 디버깅이 쉽고 여러 os와 호환이 되므로 접근성이 뛰어남

 Php

 Professional Hypertext Preprocessor의 약자로 APM의 P에 속합니다. 
   C를 기반으로 하는 스크립트 언어이며 빠르고 간단하다.개인용도로 개발되어 복잡한 시스템에서 사용하기에는
     무리인 단점이있다. 예전언어들에 비해 간결해진 코드로인해 인기를 얻은후 더욱 개성있는 
       스크립트언어들이 생겨나게 됨. php는 윈도우, 리눅스, 환경에 상관없이 컴파일만 하면 소스수정 하지않아도 
          안정적으로 동작가능 비용이 또한 무료임

	   

 Python

   python의 가장 큰 장점은 개발 속도라고 할 수 있다.javascript와 마찬가지로 인터프리터형이라서 편집-실행-디버깅 속도가 엄청 빠르기 때문에 생산성 증대가 된다. 즉 같은 시간 동안에 더 많은 코드를 만들 수 있으면서 동시에 전체 코드 길이는 짧다는 것이다.
  문법을 무척이나 간단하게 사용하면서도 모듈이나 클레스, 예외, 동적데이터형과 
형변환을 고수준으로 지원하며 많은 플랫폼에서 동작가능. 
객체지향이면서 안정적이고 상위 호환적이며, 은닉이 가능하고 이식성이 좋다고 합니다. 
멀티 스레드기반 개발을 해야할 때 Python의 성능 저하는 크나큰 치명점.
인터프리터언어이기때문에 실행속도는 C에 비해 느림. python 역시 무료

      Ruby

   Ruby는 일본에서 개발된 동적 객체지향 스크립트언어. ruby개발자 마인드가 스트레스없는 쉬운 프로그래밍을 구현하기 위해 개발된 언어기 때문에 인간 중심으로 설계되었다고 함 이식성이 높아서 유닉스 맥 윈도우 도스 등 호환가능 운영체제가 많음.
 ruby는 오픈소스로서 유저스스로가 개발하여 서로 공유를 가능 그렇기 때문에 사용자에게 맞는 루비를 숙지하여 사용하면  된다는데, perl이라는 언어가 읽는부분이 힘들다는 이유로 개발되어져서, perl보다 단순하고 외우기가 쉬움. python과 비교하자면 python에 비해 정형화된 서식을 요구하지 않다는 점이 차이점이다.
