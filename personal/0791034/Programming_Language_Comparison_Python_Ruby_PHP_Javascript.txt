Python, Ruby, PHP, Javascript 각 언어의 장점을 5가지씩 정리 및 비교


Python
1. Python은 플랫폼 독립적이며 인터프리터식이다. 인터프리터(해석기)는 프로그래밍 언어의 소스 코드를 바로 실행한다고 한다. 개발단계에서는 프로그래머가 한 번에 적은 코드를 추가하길 원하는 경우가 많은데, 올바르게 작동하는지 빠르게 알아볼 수 있기 때문에 개발용 프로그래밍 언어로서 많이 쓰인다.
2. Python은 고급 프로그래밍 언어이다. 다른 언어들에 비해서 코드가 단순하다. 때문에 원래 교육용으로 설계된 프로그래밍 언어이기 때문에 프로그래밍에 해박한 지식이 없는 사람도 배우기가 쉽다.
3. Python은 객체지향적 언어이고, 모든 명시하고 있는 것들이 객체로 취급이 된다. 따라서 Python은 Introspection기능을 제공하여 객체들의 정보를 검색, 관리, 편집이 쉽다.
4. 코드수가 C, C++, Java 같은 언어들에 비해 1/3정도이고, 개발시간도 훨씬 더 단축된다. 
5. 비영리의 파이썬 소프트웨어 재단이 관리하기 때문에 무료이다. 개발자나 개발회사들의 경제적 부담감을 덜어주기 때문에 사람들이 많이 찾는 언어이다. 


Ruby
1. 기본적인 문법만 이용하더라도 범용 스크립트 언어로 활용할 수 있다.
2. 순수한 객체지향언어로서 모든 데이터가 객체이다. 데이터타입의 일관됨으로 인해 문법이 간단하고, 코드 작성시 깔끔하다. 코드수가 기존언어들에 비해 짧고 개발시간도 큰폭으로 단축된다.
3. 메타프로그래밍을 제공하는데, 기계적이고 반복적인 코딩을 자동화하여 사용자에게 편의성을 제공한다.
4. Ruby는 Perl의 일부를 계승하고 있는데, 그로인해 텍스트 처리에 뛰어나다.
5. 이식성이 높다. 유닉스, 리눅스, 마이크로소프트 윈도, 맥 오에스 텐, MS-DOS, OS/2, 아미가 등 많은 플랫폼에 이식되어 있다.

PHP
1. 웹프로그래밍에 특화되어 있는 스크립트언어로서 주로 리눅스 웹서버에서 사용된다. 널리 사용되는 리눅스 서버에 쓰이는 언어가 PHP라는 사실만으로도 가치가 있다.
2. C언어를 기반으로 하여 만들어졌기 때문에 페이지가 열리는 속도가 빠르다.
3. HTML을 다루는 목적으로 만들어졌지만, image나 PDF, 플레쉬 등을 만들어 출력할 수 있어서 사이트나 게시판을 움직이는 이미지들을 간단하게 만들 수 있다.
4. PHP는 MySQL과 쉽게 연동할 수 있어서 DB를 다루는데 있어서 탁월함을 보인다.
5. 윈도우, 유닉스, 리눅스 같이 주요한 운영체제에서 독립적으로 개발이 가능하다.

Javascript
1. javascript를 이용하면 html만으로는 제공하기 어려운 역동적인 홈페이지를 제공할 수 있다.
2. 클라이언트 자체에서 실행하는 언어이다.
3. 스크립트 언어로서 코드를 즉시 확인하여 처리가 가능하다.
4. 윈도우, 유닉스, 리눅스 같이 주요한 운영체제에서 독립적으로 개발이 가능하다.
5. JAVA언어의 문법보다 쉬우며, 명령어의 수가 많지 않고, 내장함수와 객체를 사용하기 때문에 프로그래밍이 쉽다.
