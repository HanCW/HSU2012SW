github의 장점

1. 수정한 코드를 프로그래머 각자의 로컬 저장소에 올린다.
- 각자의 저장소를 가지기 때문에 충돌 위험이 적다.
- 네트워크 연결 환경이 좋지 않아도 저장소에 수정 내용을 반영할 수 있다.
- 작업을 마친뒤에 마스터 저장소에 올릴 수 있다.
2. 저장소의 역할 뿐만 아니라 프로그래머 간의 소셜 네트워크를 형성한다.
- 소스 코드로 프로그래머 간의 소통이 가능하다.
- 활동 내용이 모두 기록되기 때문에 작업한 내역을 쉽게 파악할 수 있게 된다.


github 사용하며 느낀 점

1. 버전관리시스템을 써본 적이 없어 굉장히 생소했다.
2. GUI에 익숙하기 때문에 명령어를 사용하는 것도 어렵게 느껴졌다.
3. 과제를 올리면서 add, commit, push에는 익숙해진 것 같다.
4. 버전관리시스템이란 것이 어떤 것이고 왜 쓰이는지 알게 되었다.
5. 파일분할도 익숙하지 않은 나에겐 조금 부담이 되는 시스템이라고 생각했다.
6. 간단한 예제에서 벗어나 여러 파일로 구성되는 프로그램을 만들 때 다시 사용해봐야겠다는 생각을 했다.
