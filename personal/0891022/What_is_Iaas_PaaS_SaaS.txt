클라우드 서비스는 다음과 같은 세 가지 유형으로 분류할 수 있습니다. 
Iaas - Iaas란 Insfrastructure, 예를 들어 서버, 저장공간, 네트워크등을 서비스로 제공하는 것입니다.
       Iaas를 서비스 하기 위해서는 IDC라는 서버, 스토리지, 네트워크 장비들이 필요합니다. 이런것들로 클라우드 환경을
       만들어 사용자들이 가상공간의 자원을 사용할 수 있게 합니다. 지금까지는 저장공간이 부족하면 물리적으로 데이터를
       저장할 공간을 필요로 했는데 Iaas 서비스로 가상공간에 저장할수 있게 되었습니다. 서버도 또한 물리적인 추가없이
       가상환경으로 늘릴 수 있습니다.
       국외 : 아마존 웹서비스 / 국내 : KT Ucloud

Saas - Saas란 소프트웨어 서비스를 제공하는 것입니다. 소프트웨어의 기능 중에 유저가 원하는 기능만을 서비스로 배포해
       이용이 가능하도록 한 소포트웨어 배포형태입니다. 기존에도 sw를 빌려쓰는 ASP라는 것이 있었지만 ASP서비스는 SW를
       획일화시켜 배포했기 때문에 각 사용자의 환경에 맞지 않는 경우도 있었습니다. 그러한 형태에서 발전한 것이 이 Saas서비스이며
       기능과 환경을 사용자에게 맞출수 있다는 점이 장점입니다.  
       국외 : 구글 Docs / 국내: 트윈캠프

Paas - Pass란 Saas에서 확장한 방식으로 sw가 아닌 플래폼을 제공하는 서비스입니다. Saas를 사용할 때는 다른 어플리케이션들과 부딪칠
       염려가 있기 때문에 Saas를 제공하는데서 openAPI 형태로 통합할 수 있는 플랫폼을 제공합니다. 개발을 위한 플랫폼 구축할 필요 없고
       필요 기능들을 클라우드 상에서 쉽게 빌려쓸수 있습니다. 이렇듯 하나의 서비스가 아닌 통합의 서비스를 제공하는 것이 Paas 입니다.
       국외 : 구글 AppEngine / 국내 : 네이버 openAPI
