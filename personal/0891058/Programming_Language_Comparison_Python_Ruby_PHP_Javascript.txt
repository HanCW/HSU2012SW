Python
1988년 네덜란드 귀도 반 로섬(Gido van Rossum)에 의해 처음 만들어졌으며 파이썬 소프트웨어 제단에서 관리 하고 있으며
라이센스가 거이 무료와 다름없는 고수준의 언어입니다.
저수준의 언어에 비해 문제를 넓은 시야로 발라보고 전반적인 이해를 통해 문제 해결에 집중할 수 있습니다.
또 이해하기 쉽습니다. 파이썬 문법은 직관적이고 이해하기 쉬우므로 더 빨리 배워서 개발을 시작할 수 있습니다.
그리고 생산성이 높습니다. 그래서 소프트웨어 개발과 유지보수에 들어가는 노력을 절감할 수 있으므로 그만큼 이득입니다.
풍부한 라이브러리뿐만 아니라 C에 있는 기능을 붙여 쓸 수 있고 반대로 파이썬 기능을 C에 사용할 수 있는 접착성이 우수합니다.
그리고 널리 사용되기 때문에 개발자 입장에서는 한정된 시간 안에 어떤 기술에 시간을 투자할 것인지 정하기 위해서는 
얼마나 많은 수요가 있는지 생각하기 때문에 널리 사용되는게 중요합니다. 구글의 3대 개발언어로 채택되면서 더욱 유명세를 타게 되었습니다.

Ruby
Ruby는 펄의 유연함과 파이썬의 객체지향성 보다 강하하고자 하는 의도로 마츠모토 유키히로에 의해 1993년 개발된 프로그래밍 언어로 1995년 첫 공개버전이 나왔다.  
웹 애플리케이션 프레임 워크 'Ruby on Rails' 로 Ruby의 유연한 문법과 뛰어난 확장성 특징으로 크게 주목을 받게 되었다.
Ruby에는 통상적인 명칭으로 '진정한 객체지향언어'(객체가 아닌 자료형은 존재하지 않고 모든 것이 객체이다), 
'보다 나은 펄'(문자열을 간단히 다룰 수 있는 다수의 기능과 강력한 텍스트 처리기능 을 실현 또 유닉스 호환 시스템에 있어 C언어 라이브러리를 랩핑한 라이브러리 지님),
'동작하는 의사코드'(코드의 표기가 간결해 친숙해지기 쉽고 Ruby 인터프리터를 이해시키기 위해 다양한 주문을 걸 필요 없이 
본질적인 처리만 기억하면 알고리즘이나 소프트웨어 개발의 보편적인 개념을 설명하는 사람들은 코드 예문으로 Ruby를 이용하기 좋다),
'ALGOL의 탈을 쓴 Lisp'(ALGOL계 프로그래머에게 친숙하게 설계되었다. 하지만 Lisp으로 부터 많은 사고방식을 계승했다.) 
라 하는데 이 명칭들이 모두 Ruby의 특징이다.

PHP
PHP(Professional HTML Preprocessor)는 가장 많이 사용되는 웹언어(PHP,JSP,ASP) 중 하나로 많은 함수를 기본적으로 제공하고 프로그래밍의 정확성과 생산성을 동시에 
이룰 수 있도록 제공뿐만 아니라 초보자가 쉽게 접근하여 배우기가 쉽기 때문에 PHP를 이용하여 웹 사이트를 개발하는 개발자들이 많이 증가 추세다.
PHP는 운영체제로 윈도우뿐만 아니라 리눅스, 유닉스계열에서도 사용할 수 있고 PHP로 작성된 프로그램은 웹서버의 운영체제가 바뀌더라도 소스를 거의 수정하지 않고 그대로 사용할 수 있다.
또 PHP는 다양한 데이터베이스를 지원하지만 그중 MySQL이 가장 연동이 잘 맞고 또 무료로 제공되기 때문에 널리 사용된다. 
그리고 APM(Apache, PHP, MYSQL)을 하나의 패키지로 제공되어 따로 따로 설치하여 환경 설정하는 작업의 부담을 덜어준다.
그와 다르게 ASP(Active Server Pages)는 주로 윈도우 운영체제에 익숙한 사용자들은 선호하지만 리눅스나 유닉스계열 운영체제 에서는 사용하지 못한다.
또 JSP(Java Server Pages)는 Java의 객체지향에 대한 충분한 이해와 응용이 필요한 하다.
JSP와 ASP는 오라클이나 MS-SQL 또는 다른 데이터베이스 연동하기 위해 까다로운 설정과정이 반드시 필요하지만 PHP는 MySQL 연결을 위해 몇 줄의 코드 작성으로 쉽게 설정할 수 있다.

Javascript
Javascript란 넷스케이프(Netscape)사와 썬 마이크로 시스템(SunMicroSystem)사가 개발한 웹 문서 작성 언어이며
Java Applet, Active-X Control, Browser Plug-in, ServerScript 등 기타 웹 객체들과 연동이 가능한 언어로 Javascript code를 통해 모든 웹페이지의 요소들을 접근할 수 있도록 지원한다.
또, 객체를 처리할 수 있도록 이벤트 함수를 지원한다.
Javascript는 컴파일 과정을 거치지 않고 브라우저에 실행되는 인터프리터 방식의 언어이고 객체지향적 특성을 가지고 있지만 일관성이 부족하며 진정한 객체지향 언어로 보기 어렵다.
Javascript는 이미 성숙한 언어로 여러 기능을 두루 갖추고 있으며 충분한 보안 기능을 제공한다.
마지막으로 다른 프로그래밍 언어와 달리 XML이나 HTML 같은 다른 언어의 내부에 삽입되는 경우가 대부분인 특징이 있다
