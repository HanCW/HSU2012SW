GitHub는 이번 학기에 처음 접하게된 프로그램이었다.
GitHUb는 팀프로젝트로 각자가 맡은 소스코드를 한곳에 모으고, 버전 관리를 해주는 시스템이다.
각자가 작성한 소스코드는 한곳에 모아 관리 되어지는데, 다른 개발자가 수정한 파일을 받을수도 있고
내가 수정한 파일도 올려 다른 사용자가 최근 수정된 파일을 받을 수도 있다. 

학교에서 간략한 사용 설명을 듣고 집에 와서 시도 했지만, 결코 만만치 않았다.
사용 설명이 있는 파일도 받아서 GitHub의 명령어에 대한 설명을 읽어 보았지만
막상 내가 직접 따라 해보려 하니 막히는 부분도 나오고, 해결하려니 어려웠다.
생소한  fork, push, origin, master, fullrequest와 같은 단어들과, 
알 수 없는 오류들로인해 막막했다.

네이버나 구글에서 해당 오류를 찾고, 동작 원리를 읽어가며 어렵게  fork를 하였고, push를 할 수 있게 되었다.
과제 제출을 통해 push기능을 잘 알게 되었지만, 나머지 기능에 대해선 아직도 어색하다.

GitHub를 실행해도 CMD창 에서 리눅스 명령어와 비슷한것을 입력 할뿐 변동 사항과 진행 사항이 시각화 되어 있지 않아 
습득하는데 힘들게 느껴졌다.
아직 우리나라에선 아는 사람만 사용하는지 블로그의 설명도 쉬운걸 찾기 어려웠고, 구글링을 통해 얻은 자료는 모두 영문 이었다.

progit 문서를 읽어 보았지만, 단편적인 예만 나와있었고, 동작에 대한 이해를 알기엔 부족했다.
조금 더 쉬운 문서나, 블로그의 포스트가 있었으면 GitHub에 대해 조금 더 빨리 친해 질 수 있었을것 같았다.
또한 CMD창에서 명령문으로만 진행하는 것에 대해 조금 아쉬웠다.
GUI의 GitHub에 대한 설명은 찾아보기 더 힘들었고, 전에 시도 했다가 README 파일을 지워버려 다시 fork하는 경험도 하였다.

GitHub에 대해 정확하게알고 Word 사용 하듯이 사용 할줄 알면 굉장히 편하고 좋은 시스템이란것은 알겠지만,
첫 유저가 GitHub에 대한 진입 장벽이 너무 높아 처음 사용자에겐 너무 어렵게만 느껴져서 
처음 사용자에게 맞는  쉽고 체계적인  설명이 있었으면 하는 아쉬움이 남았다.
